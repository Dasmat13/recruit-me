About My Journey

I started my journey exploring algorithmic logic and competitive programming, which built my foundation in Data Structures and Algorithms. Soon after, I fell in love with server architectures and building distributed APIs. I build system backends that aren't just fast but easily maintainable and fully monitored.

Over the past few years, I've designed cloud setups, configured complex pipelines, and contributed to key open-source projects including Kubernetes tools and development utilities.

My core engineering rules are simple:

  • Clear over Clever: Write readable, simple code before optimizing prematurely.
  • Observability is Mandatory: If a system isn't monitored with metrics (like Prometheus/Grafana), it's broken.
  • Automate Everything: CI/CD, lint checkers, and tests save thousands of engineering hours.
  • I once optimized a script to reduce dataset annotation times by 80%.
  • My terminal theme is dark mode exclusively (light terminal users scare me).
  • When not writing code, you can find me reviewing Kubernetes controllers or solving leetcode queries.
